# AI Development Index (AIDI)

**AI能力的"GDP"——一个可检验、可追溯、可校准的全球AI发展定量追踪系统。**

不是拍脑袋的评分，不是大模型公司的PR稿汇总。是**从物理隐喻出发，用数学工具建模，用真实数据注入，用后见之明校准**的工程化指数。

---

## 为什么需要这个指数？

2022年12月至今，AI领域发布了200+个模型，API价格降了100倍，Agent能力从零到商用。但我们缺一个**统一的衡量框架**来回答三个问题：

> 1. AI整体到底变强了多少倍？
> 2. 现在发展有多快？
> 3. 中国和美国差距到底多大？

AIDI的使命就是回答这三个问题——用可复现、可检验的方式。

---

## 设计原理

### 第一性原理：从物理隐喻出发

AI发展不是单一维度的指标。我们将其建模为**三维空间**：

```
┌─────────────────────────────────────────────────────────┐
│                    AIDI 三维模型                          │
│                                                         │
│  v3 能力空间 (AIC)         v4 速度空间 (DVI)               │
│  ┌───────────────────┐   ┌───────────────────┐           │
│  │ 智力(LLM推理)      │   │ 硬件(GPU/HBM)     │           │
│  │ 感官(多模态)       │   │ 效率(量化/蒸馏)   │           │
│  │ 行动(Agent)       │   │ 定价(API成本)     │           │
│  │ 编程(代码)        │   │ 渗透(采用率)      │           │
│  │ 知识(RAG/向量库)   │   └───────────────────┘           │
│  │ 生态(基建/成本)    │   发展有多快 → DVI + HMM状态       │
│  └───────────────────┘                                    │
│  有多强 → AIC + 交互效应                                  │
│                                                         │
│  CN-US Parity (差距)：纯能力×0.5+性价比×0.25+开源×0.15+创新×0.1│
└─────────────────────────────────────────────────────────┘
```

**为什么用三维而不是一个数？** 因为AI发展是"能力的解锁×效率的提升×普及的速度"三股力量的乘积效应，单个数会丢失信息。

### 数学基础

#### AIC (AI Capability) — 能力总量

```
AIC = 六维能力向量平均值 + 跨维度交互效应

基础项:       base = Σ(维度分) / 6
交互项:       synergy = Σ(s_ij × min(vi,vj)/1000 × vi×vj/1e6 × 500)

其中 s_ij 是维度i和j的协同系数（由格兰杰因果分析确定）
```

**设计依据**: AI能力不是各维度的简单加法。一个智力高但行动力为零的AI做不了事；一个智力高且行动力高的AI能做大事。交互项用**弱维度约束×乘积效应**来建模这种协同——链子的强度取决于最弱的一环。

#### DVI (Development Velocity) — 发展速度

```
DVI = 硬件(0.25) + 效率(0.25) + 定价(0.25) + 渗透(0.25)

发展速度 = 供应(硬件+效率) + 成本(定价) + 需求(渗透)
```

**设计依据**: 等权是最无偏的数学选择（无先验信息时的最大熵分布）。格兰杰因果检验显示四维度高度耦合（r>0.98），说明AI发展是一个自增强的飞轮：硬件→效率→定价→渗透→更多投资→硬件。

#### AIDI (Development Speed) — 发展加速度

```
AIDI(t) = AIC(t) - AIC(t-1)    （每半月能力增量）
```

物理类比：AIC是位置，AIDI是速度。2022-12基线设定AIDI=100km/h（你指定的初始速度）。

#### HMM状态检测 — 发展阶段识别

基于DVI增速，检测四个隐藏状态：
- **爆发期**: DVI增速 > 100 → 多维度共振（如2026Q1 Capybara+GPT-5.4+DeepSeek V4同时爆发）
- **加速期**: DVI增速 40-100 → 单一维度突破
- **平台期**: DVI增速 10-40 → 能力持续积累但无重大突破
- **减速期**: DVI增速 < 10 → 供给侧瓶颈

**设计依据**: AI发展是非线性的——有爆发期也有平台期。线性模型无法捕捉这种相变。HMM用隐藏状态来建模"发展阶段"的切换，比简单回归更符合AI发展的实际历史。

#### PSO权重优化 — 假设最少的参数设计

粒子群优化算法在30个智能体中搜索最优维度权重，目标函数为最小化预测误差。结果显示等权分配是最稳健的选择（任何试图偏向单一维度的权重都会在其他时期产生回弹误差）。

#### 卡尔曼平滑 — 剔除周期噪声

AI发展速度测量有周期噪声（财报季、学术会议季、新品发布季）。卡尔曼滤波器用过程噪声(0.05)和测量噪声(0.15)参数，将真实趋势与周期波动分离。

## 校准机制：后见之明的合法性

这是AIDI区别于其他AI指数的核心设计。

**问题**: 2023年o1刚出现时，没人知道Agent能力会发展成什么样。当时评的分必然偏低。

**解法**: 每次校准允许回溯修正历史评分，并记录修正原因。

```
data/calibration_log.json
├── 每次修正记录: {date, trigger, changes: [{period, dim, old, new, reason}]}
├── 可追溯: 任何时期的分数都知道"当时怎么评的"和"后来怎么修正的"
└── 永不删除原始值: revisions数组累积而非覆盖
```

**科学依据**: 这是**贝叶斯更新**在时间序列上的应用。每次新证据（新模型的发布）不仅更新当前期，也更新我们对历史事件的理解。就像一个天文学家发现新天体后，会重新校准望远镜——不是之前的观测错了，而是有了新的参考系。

## 数据层架构（引擎不可知）

```
data/
├── raw/YYYY-MM-DD/_search.json  ← 原始搜索数据 (不可变)
│                                   86期, 每含模型发布/基准/事件
│
├── curated/
│   ├── dim_scores.json          ← 六维能力评分 (可校准)
│   ├── calibration_log.json     ← 每次校准记录 (可追溯)
│   └── complete_snapshot.json   ← 全量快照索引
│
├── benchmarks/                   ← 外部基准 (Arena Elo/MMLU/GPQA)
│
engine/
├── aidicore_v3.py               ← 能力指数引擎 (读dim_scores.json)
├── aidicore_v4.py               ← 发展速度引擎 (PSO/HMM/Kalman)
└── period_search.py             ← arXiv自动搜索 (辅助)
```

**设计原则**: 数据与引擎解耦。v3/v4引擎读取相同的数据文件。未来升级到v5/v6只需增加引擎代码，不需要修改历史数据。

## 使用

```bash
pip install -r requirements.txt
python engine/aidicore_v3.py   # 能力指数
python engine/aidicore_v4.py   # 发展速度指数
```

**数据获取**: 零成本, 纯免费API (arXiv/GitHub/feedparser)

**校准**: 新模型发布后运行校准脚本, 自动记录到calibration_log.json
